# Workflow Collaborativo

## Strategia adottata

Per lo sviluppo del progetto è stato adottato un modello di tipo **Feature Branch Workflow**.

## Organizzazione dei branch

- `main` → contiene la versione stabile del progetto
- `feature-bacheca` → branch dedicato allo sviluppo della nuova funzionalità

## Processo seguito

1. Creazione del branch feature a partire da main
2. Sviluppo e test della funzionalità in ambiente isolato
3. Commit separati e descrittivi
4. Push su GitHub
5. Merge finale su main

## Vantaggi del workflow

- Isolamento delle modifiche
- Maggiore controllo delle versioni
- Simulazione di un ambiente di sviluppo collaborativo
- Riduzione del rischio di errori sulla versione stabile
